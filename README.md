# 游戏数据实时监控系统

## 📋 项目简介

本项目是一个基于 Vue 3 开发的游戏数据实时监控系统，面向游戏运营团队，提供服务器状态、在线玩家、收入数据等核心指标的实时监控、可视化分析和智能告警功能。系统采用前后端分离架构，前端通过 WebSocket 实现实时数据推送，集成 ECharts 进行数据可视化展示，支持多角色权限控制和自定义仪表板配置。

## ✨ 功能特性

### 1. 实时监控模块
- **服务器状态监控**：CPU使用率、内存占用、网络延迟、在线人数实时展示
- **游戏业务指标**：DAU、PCU、收入数据实时更新
- **多服务器集群监控**：以列表形式展示多台服务器运行状态

### 2. 数据可视化模块
- **趋势分析图**：在线玩家趋势、收入变化曲线
- **仪表盘展示**：关键指标卡片、实时数据仪表盘
- **图表联动**：图表间数据联动和缩放查看

### 3. 智能告警模块
- **多维度告警**：阈值告警、趋势告警、异常检测
- **告警方式**：邮件、短信、系统通知
- **告警历史**：记录查询和统计分析

### 4. 数据分析模块
- **玩家行为分析**：留存率、付费率、活跃度
- **服务器性能分析**：负载趋势、资源使用率
- **数据导出**：支持 Excel、CSV、PDF 格式导出

### 5. 权限管理模块
- **多角色权限**：管理员、运营人员、开发人员、观察员
- **按钮级控制**：不同角色看到不同的操作按钮
- **路由权限**：不同角色可访问的页面不同
  
### 6. 数据持久化模块
- **MySQL 数据库存储**：游戏指标历史、服务器状态、告警记录
- **历史数据查询**：支持按时间范围查询历史趋势
- **数据备份**：支持导出历史数据
  
  ### 7. WebSocket 实时通信
- **双向通信**：后端主动推送数据到前端
- **断线重连**：自动检测连接状态，5秒后自动重连
- **心跳检测**：保持连接活跃
  

## 🛠️ 技术栈

| 分类 | 技术选型 | 说明 |
|------|----------|------|
| **前端框架** | Vue 3 + Composition API | 使用 `<script setup>` 语法，实现响应式数据 |
| **状态管理** | Pinia | 替代 Vuex，更简洁的 API 设计 |
| **路由管理** | Vue Router 4 | 实现页面导航和权限控制 |
| **UI组件库** | Element Plus | 提供丰富的 PC 端组件 |
| **数据可视化** | ECharts 5 | 支持多种图表类型和交互 |
| **实时通信** | WebSocket | 建立双向持久连接 |
| **HTTP客户端** | Axios | 处理 RESTful API 请求 |
| **构建工具** | Vite | 极速冷启动和热更新 |
| **日期处理** | Day.js | 轻量级日期处理库 |
| **导出工具** | XLSX + jspdf | 支持 Excel、CSV、PDF 导出 |
| **后端框架** | Node.js + Express | RESTful API 服务 |
| **数据库** | MySQL 8.0 | 数据持久化存储 |
| **WebSocket 服务** | ws 库 | 原生 WebSocket 实现 |

## 🚀 快速开始

## 环境要求
**• Node.js 18+**  
**• npm 8+ 或 yarn 1.22+**
*• MySQL 8.0+**

## 安装步骤
### 1. 实时监控模块
```bash
# 进入数据库目录
cd database

# 执行 SQL 脚本
mysql -u root -p < init.sql
```
### 2. 启动后端服务
```bash
# 进入后端目录
cd backend

# 安装依赖
npm install

# 启动后端
npm start
```
### 启动成功后会看到：
```bash
🚀 HTTP 服务启动: http://localhost:3001
🔌 WebSocket 服务启动: ws://localhost:3002
✅ 数据库连接成功
```
### 3. 启动前端服务
```bash
# 进入前端目录（新终端）
cd frontend

# 安装依赖
npm install

# 启动前端
npm run dev
```
### 4. 访问系统
开浏览器访问：http://localhost:8080
